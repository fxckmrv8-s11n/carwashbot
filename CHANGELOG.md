# CHANGELOG.md — Редизайн CarWash Cloud под концепт «Glass / Orb»

Полная миграция дизайна сайта и Telegram Mini App на новую визуальную
концепцию из `main-page-concept.html`: тёмный фон, плавающие цветные
орбы, полупрозрачные glass-карточки с блюром, анимированная градиентная
hero-карточка. Выполнено в 6 этапов, каждый — отдельный проверяемый шаг
без изменения бизнес-логики и backend.

Backend (`server.py`, `bot.py`, `config.py`, остальные `.py`-модули) —
**не изменялся**. Все правки — только `webapp/static/*`.

---

**Файлы:** `webapp/static/site-theme.css`, `webapp/static/index.html`,
новый `DESIGN_SYSTEM.md`.

- Добавлены новые CSS-токены: `--glow-a/b/c` (цвета орбов), `--glass`,
  `--glass-hover`, `--glass-border`, `--text-dim`, `--text-faint`, `--good`,
  `--radius-sm/md/lg/xl`, `--blur-glass`, `--blur-nav`.

  `--accent`/`--accent-b` → `#6A8BFF`/`#A24BFF` (было `#7C6CF6`/`#4E9BFF`).
- Новые переиспользуемые компоненты (одинаковые классы на сайте и в
  mini-app): `.orb-bg`/`.orb` (плавающие фоновые пятна с анимацией),
  `.glass-card`, `.hero-glass` (анимированный градиент), `.pill.ok/.wip`,

- Старые имена переменных (`--surface-*`, `--accent-tint` и т.д.)

  написанных страниц.

## Этап 2 — Mini App: экран «Главная»

**Файл:** `webapp/static/index.html` (функция `renderHome()`).

- Экран полностью пересобран по макету из `main-page-concept.html`:
  шапка с приветствием по времени суток + градиентный аватар-инициалы,

## Этап 3 — Mini App: остальные экраны

**Файл:** `webapp/static/index.html`.

Общие переиспользуемые классы (используются на Кассе, Машинах, Финансах,
Отчётах, Сотрудниках, Настройках, Истории) переведены на glass/blur —
эффект автоматически применился ко всем этим экранам без правки
render-функций каждого из них:

- `.card` / `.row` — сплошной фон → полупрозрачный с блюром, радиус
  16px → 22px.
- `.op-list` / `.op-row` — аналогично.
- `.segmented` / `.seg` — glass-таблетка, активный сегмент — акцентный
  градиент.
- `.tile` (Отчёты) — glass.
- `.sheet` (нижние контекстные меню) — сильный блюр вместо сплошной
  заливки.
- `.home-stat-box` (Моя смена/График), `.rstat-box` (Отчёты) — glass.
- Тень `.btn` пересчитана под новый акцентный цвет.

Намеренно не тронуто: внутренняя вёрстка форм добавления машины, donut-
диаграмма в Отчётах, свайп-карточки кассы — риск сломать жесты без
тестирования на устройстве.

  `#08080B`) в переключателе темы mini-app.

## Этап 5 — Сайт: страницы по одной

**Файл:** `webapp/static/site-theme.css`.

Важная находка в процессе: каждая из 10 страниц сайта хранит собственную
устаревшую инлайн-копию каркаса (`:root` со старыми iOS-цветами,
дублирующиеся `.sidebar`/`.panel`/`.btn` и т.д.) — легаси с ранних версий
проекта. Поскольку `site-common.js` подключает `site-theme.css` через
`document.head.appendChild(...)` (т.е. физически ПОСЛЕ инлайн-`<style>`
каждой страницы), при равной специфичности побеждает последнее по
DOM-порядку правило — и `site-theme.css` уже перекрывал все инлайн-копии
начиная с Этапа 4. Проверено сравнением (`diff`) стилей всех 10 страниц:
структура идентична, отличаются только значения, не участвующие в
итоговом рендере.

Добавлены точечные глобальные правила в `site-theme.css` (с `!important`
там, где нужно перебить инлайн-правило той же специфичности) — применились
сразу ко всем страницам:

- `.pay-tag` (способ оплаты — Dashboard/Касса/Финансы/История):
  прямоугольник → пилюля.
- `table.data` (таблицы операций): hover-строка и границы ячеек — glass.
- `.tag` (статусы — Мойщики/Филиалы): пилюля.
- `.tabs`/`.tab` (переключатель периода — Отчёты): glass-таблетка,
  активный сегмент — акцентный градиент.
- `.branch-card` (карточки филиала — Филиалы/Отчёты): glass + блюр.
- Добавлен токен `--accent-gradient` в `site-theme.css` для переиспользования.

## Известные ограничения

- 10 HTML-страниц сайта содержат мёртвый код — устаревшую инлайн-копию
  каркаса, которая не влияет на итоговый вид (перекрывается каскадом), но
  формально это дублирование. Полная чистка — отдельная задача по
  рефакторингу кода, не требуется для визуального результата этого ТЗ.
- Внутри mini-app не тронуты: форма добавления машины (чипы услуг),
  donut-диаграмма в Отчётах, свайп-карточки кассы — можно перевести на
  glass отдельным шагом при необходимости, вместе с тестированием
  жестов на реальном устройстве.
