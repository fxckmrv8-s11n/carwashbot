# NEW_STRUCTURE.md — Этап 2: Новая архитектура Mini App

## 1. Принцип

Mini App остаётся ОДНИМ файлом `index.html` (SPA) — это правильно для
Telegram Mini App (быстрый старт, initData-авторизация, нет лишних
переходов/перезагрузок). Меняем не файловую структуру, а:
1. навигацию — с плоских 5 вкладок на группированное нижнее меню + разделы
   верхнего уровня, зеркалящие сайдбар сайта;
2. визуальный слой — новая тема на основе `site-theme.css` (тот же
   Revolut-style: `--accent:#7C6CF6→#4E9BFF`, поверхности `--surface-0..3`,
   градиентные акценты, мягкие тени), адаптированная под мобильный экран;
3. состав экранов — «Финансы» и «Филиалы» выделяются в собственные разделы
   (сейчас размазаны/отсутствуют), у каждого раздела — свои фильтры/поиск/
   пустые состояния.

## 2. Новая навигация (нижнее меню, 5 слотов — лимит для мобильного таб-бара)

Как на сайте (`NAV` в site-common.js), но сгруппировано под 5 иконок с
подразделами внутри (те, что не помещаются — уходят в «Ещё»/профиль):

```
[Главная] [Касса] [Машины] [Финансы] [Ещё]
```

- **Главная** — дашборд дня: ключевые показатели смены (выручка, кол-во
  машин, разбивка по оплате), быстрые действия («Новая машина», «Закрыть
  смену»), лента последних событий. Заменяет нынешний renderHome-монолит,
  разгружая его до сводки + переходов.
- **Касса** — то, что раньше было основной частью renderHome: список машин
  за смену, добавление/редактирование, свайп-карточки, поиск/фильтр по
  статусу оплаты.
- **Машины** — переработанный renderAllCars: полная база машин (все смены),
  поиск по номеру, история по машине, лояльность/скидки — здесь, а не
  спрятаны в контекстном меню.
- **Финансы** — НОВЫЙ раздел (сейчас отсутствует как отдельный экран):
  расходы/доходы отдельно от кассы дня, с фильтром по периоду, разбивкой по
  категориям — зеркалит `finance.html`.
- **Ещё** (bottom-sheet или отдельный экран) — Отчёты, Мойщики/Сотрудники,
  Моя смена, История изменений, Филиалы (только для владельца), Настройки,
  Профиль. Здесь не флэт-список, а карточки-группы («Управление командой»,
  «Аналитика», «Система»).

## 3. Экраны и что в каждом нового относительно текущего index.html

| Раздел | Источник функций | Что добавляем |
|---|---|---|
| Главная | часть renderHome | сводные карточки-метрики, лента событий, быстрые действия крупными кнопками |
| Касса | renderHome (кассовая часть), renderHomeSwipeCard | поиск/фильтр по статусу оплаты, явное состояние «смена открыта/закрыта» |
| Машины | renderAllCars, renderCarsList | поиск по номеру, сортировка, вынесенная карточка лояльности (сейчас — глубоко в контекстном меню) |
| Добавить машину | renderAdd | без изменений по функциям, обновляем визуал (карточки услуг, чипы) |
| Финансы (новый) | API `/api/expense`, `/api/income` (уже используются в renderHome) | отдельный экран: список операций, фильтр по периоду, итоги — по образцу `finance.html` |
| Отчёты | renderReports, renderDayReport | карточки метрик вместо текстовых alert-попапов (сейчас `tg.showAlert` с текстом — заменить на экран) |
| Сотрудники | renderWorkers, renderEmployees, renderScheduleWeek | объединить в один раздел с подвкладками (Список / Смены / Моя смена) |
| Моя смена | renderMyStats | остаётся, доступность по `ME.is_employee` — без изменений |
| История | renderHistory | фильтр по типу события |
| Филиалы (новый) | сайт: `branches.html`, backend уже есть | добавляем экран для владельца — сейчас в mini-app такого раздела нет вовсе |
| Настройки | renderSettings | группировка (Доступы / Филиал / Тема) вместо плоского списка |

## 4. Компоненты, которые создаём/выносим в переиспользуемые

- `BottomNav` — 5 иконок + активное состояние (замена текущего `.tab` ряда).
- `MetricCard` — карточка показателя (число + подпись + тренд/иконка),
  для Главной и Отчётов.
- `SectionHeader` — заголовок раздела с действием справа (поиск/фильтр/кнопка).
- `EmptyState` — единый компонент «здесь пока пусто» с иллюстрацией-иконкой
  и CTA (сейчас пустые состояния оформлены неединообразно).
- `Skeleton`/`LoadingState` — единый загрузочный плейсхолдер вместо резкой
  подмены DOM.
- `ConfirmSheet` — обёртка над `tg.showConfirm` с текстом-подтверждением
  единого стиля.
- `Toast` — лёгкое уведомление об успехе действия (сохранено/удалено),
  дополняет `tg.HapticFeedback`.
- `SearchInput` — переиспользуемое поле поиска (Машины, Финансы, История).

## 5. Тема

Переносим CSS-переменные из `site-theme.css` в `index.html` (или выносим в
отдельный `<style>`/файл, который подключают оба интерфейса), сохраняя фирменный
стиль: тёмный фон `#08090C`, акцент `#7C6CF6→#4E9BFF`, статус-цвета зелёный/
красный/оранжевый, радиусы 11–14px, полупрозрачные бордеры, мягкие тени.
Это даёт визуальный паритет с сайтом, о котором просил заказчик.

## 6. Что НЕ меняется

- Backend API — 0 изменений в `webapp/server.py` и соседних модулях.
- Telegram.WebApp интеграция (initData, haptics, showPopup/showConfirm/
  showAlert, setHeaderColor) — остаётся, это то, что отличает mini-app от
  сайта.
- Точка входа из бота (`bot.py`) — без изменений.

## 7. План разработки (порядок реализации в Этапе 3)

1. Тема: перенос переменных `site-theme.css` → базовые стили index.html.
2. Каркас: `BottomNav` (5 разделов) + роутинг между разделами вместо
   текущего `.tab`-переключения.
3. Общие компоненты: `MetricCard`, `SectionHeader`, `EmptyState`,
   `LoadingState`, `Toast`.
4. Экран «Главная» (сводка + быстрые действия).
5. Экран «Касса» (перенос кассовой части renderHome + поиск/фильтр).
6. Экран «Машины» (перенос renderAllCars + поиск + лояльность на видном месте).
7. Экран «Финансы» (новый, на основе существующих `/api/expense`,
   `/api/income`).
8. Раздел «Ещё»: Отчёты, Сотрудники (+Моя смена), История, Филиалы
   (новый), Настройки — обновляем визуал, функции берём как есть.
9. Пустые состояния, загрузка, подтверждения — по всем экранам.
10. Финальная проверка, исправление ошибок, сборка ZIP.
