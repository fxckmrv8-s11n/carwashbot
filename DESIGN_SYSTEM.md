# DESIGN_SYSTEM.md — дизайн-концепт «Glass / Orb»

Источник: `main-page-concept.html`, присланный пользователем как эталон
нового дизайна. Цель миграции — перевести весь сайт (`webapp/static/*.html`)
и Mini App (`webapp/static/index.html`) на этот визуальный язык, не ломая
существующую функциональность и backend.

Работа ведётся ПОЭТАПНО (см. чек-лист в конце файла). Этот файл — общий
источник правды по токенам и переиспользуемым компонентам; страничные
`.md`-файлы (`NEW_STRUCTURE.md`, `MINIAPP_AUDIT.md`) фиксируют прогресс по
конкретным экранам.

## 1. Где живут токены

- **Сайт**: `webapp/static/site-theme.css` — подключается один раз через
  `site-common.js` на все страницы сайта (кроме `site-login.html`, где
  подключён напрямую `<link>`).
- **Mini App**: `webapp/static/index.html` — свой инлайн `<style>` (SPA не
  грузит `site-theme.css` отдельно), поэтому те же токены и классы
  продублированы внутри файла. При изменении палитры/компонентов — менять
  синхронно в обоих местах.

## 2. Палитра (токены)

| Переменная | Значение (тёмная тема) | Назначение |
|---|---|---|
| `--bg` | `#08080B` | фон приложения/сайта |
| `--surface-0..3` | `#0F1116 → #262832` | непрозрачные поверхности (сайдбар, инпуты) — используются там, где glass-эффект не нужен |
| `--text` / `--text-secondary` / `--text-muted` | `#F5F6FA` / `#9096A3` / `#5E6270` | текст (старые имена, используются в уже написанных экранах) |
| `--text-dim` / `--text-faint` | `rgba(245,246,250,.56)` / `.34` | текст новых glass-компонентов (из концепта) |
| `--accent` / `--accent-b` | `#6A8BFF` / `#A24BFF` | основной градиент (кнопки, активные состояния, hero) |
| `--good` / `--green` | `#34D399` | успех/оплачено |
| `--orange` | `#FFB648` | предупреждение/в работе |
| `--red` | `#FF6B8B` | ошибка/удаление |
| `--glow-a/b/c` | `#3B6BFF` / `#A24BFF` / `#FF4FA3` | цвета фоновых «орбов» и hero-градиента |
| `--glass` / `--glass-hover` / `--glass-border` | `rgba(255,255,255,.055/.09/.09)` | фон/бордер стеклянных карточек |

Радиусы: `--radius-sm:12px`, `--radius-md:16px`, `--radius-lg:22px`,
`--radius-xl:30px`. Блюр: `--blur-glass:20px`, `--blur-nav:28px`.

Светлая тема (`html[data-theme="light"]`) использует те же имена
переменных с адаптированными значениями (glass — тёмный полупрозрачный на
светлом фоне, а не белый).

Старые переменные (`--surface-*`, `--accent-tint`, `--purple-tint` и т.п.)
**сохранены** — их использует весь уже написанный HTML/JS. Новые токены
добавлены рядом, ничего не переименовано и не удалено, чтобы не сломать
страницы до того, как до них дойдёт очередь по плану.

## 3. Переиспользуемые компоненты (одинаковые классы на сайте и в аппе)

- **`.orb-bg` + `.orb.orb-a/.orb-b/.orb-c`** — фиксированный слой из 3
  размытых цветных пятен (`filter:blur(90px)`), плавно двигающихся
  (`@keyframes orb-float-1/2`). Добавляется один раз в разметку страницы,
  `pointer-events:none`, не мешает контенту.
- **`.glass-card`** — базовая полупрозрачная карточка с блюром фона
  (`backdrop-filter`). Строительный блок для списков, статов, секций.
- **`.hero-card`** — крупная карточка с анимированным цветным градиентом
  (`@keyframes hero-shift`) для главных сумм/итогов (выручка за смену,
  баланс).
- **`.pill.ok` / `.pill.wip`** — статус-бейджи (оплачено / в работе).
- **`.action-tile`** + `.sq.b1..b4` — квадратные плитки быстрых действий
  с цветными градиентными иконками.
- **`.glass-navbar`** — плавающий полупрозрачный нижний навбар (мобильный).

### Этап 5 — детали (Сайт: страницы по одной)

Важная находка: каждая из 10 страниц сайта хранит собственную инлайн-копию
общего каркаса (`:root`, `.sidebar`, `.panel`, `.stat-card`, `.btn` и т.д.) —
устаревшее легаси ещё с ранних версий проекта. Но `site-common.js`
подключает `site-theme.css` через `document.head.appendChild(...)`, что
физически помещает этот `<link>` **после** инлайн-`<style>` каждой
страницы. При равной специфичности побеждает последнее по порядку
правило — то есть `site-theme.css` **уже перекрывал** все инлайн-копии на
всех 10 страницах начиная с Этапа 4, просто это не было явно
задокументировано. Проверено сравнением (`diff`) стилей всех страниц:
структура идентична, отличаются только значения (палитра, отступы),
которые не участвуют в рендере, потому что переопределены.

Из-за этого страницы сайта уже выглядели в стиле «Glass/Orb» сразу после
Этапа 4. Этап 5 добавил точечные глобальные переопределения в
`site-theme.css` (используют `!important` там, где нужно перебить
инлайн-правило с точно такой же специфичностью и структурой селектора) —
они автоматически применились сразу ко всем страницам, которые содержат
эти компоненты:

- **`.pay-tag`** (способ оплаты — Dashboard/Касса/Финансы/История):
  прямоугольный бейдж → пилюля (`border-radius:100px`), как `.pill` в
  дизайн-концепте.
- **`table.data`** (таблицы операций — Dashboard/Финансы/История/Отчёты):
  hover-строка — `--glass-hover` вместо сплошного `--surface-2`; границы
  ячеек — `--glass-border`.
- **`.tag`** (статусы — Мойщики/Филиалы): пилюля вместо прямоугольника.
- **`.tabs`/`.tab`** (переключатель периода — Отчёты): glass-таблетка,
  активный сегмент — акцентный градиент (как `.segmented`/`.seg` в
  mini-app).
- **`.branch-card`** (карточки филиала — Филиалы/Отчёты): glass + блюр.
- Добавлен токен `--accent-gradient` в `site-theme.css` (уже был в
  `index.html`) для переиспользования в `.tab.active` и будущих правках.

Не тронуто в этом этапе намеренно: мелкие внутренние элементы форм
(`.svc-item`, `.csf-row`, `.split-row`, `.field`, `.list-row` и т.д.) —
это строки/чипы **внутри** уже стеклянных `.panel`/`.card`, их сплошной
слегка затемнённый фон обеспечивает читаемость поверх блюра (аналог
`.action-tile .sq` в концепте — сплошной элемент поверх стекла). Полная
инвентаризация всех 10 инлайн-`<style>`-блоков как технический долг
(дублирование, устаревшие переменные) вынесена в раздел «Известные
ограничения» ниже — переписывание не требуется для итогового вида, но
может быть сделано отдельно для чистоты кода.
